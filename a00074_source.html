4 # Overview
5 
6 Welcome to the documentation for the neural network potential package! This
7 repository (obtain source code
8 [__here__](http://github.com/CompPhysVienna/n2p2)) provides ready-to-use
9 software for high-dimensional neural network potentials in materials science.
10 The methodology behind the Behler-Parinello neural network potentials was first
11 described here:
12 
13 [J. Behler and M. Parrinello, Phys. Rev. Lett. 98, 146401 (2007)](https://doi.org/10.1103/PhysRevLett.98.146401)
14 
15 This package contains software that will allow you to use existing neural
16 network potential parameterizations to predict energies and forces (with
17 standalone tools but also in conjunction with the MD software
18 [LAMMPS](http://lammps.sandia.gov/)). In addition it is possible to train new
19 neural network potentials with the provided training tools.

44 from the main page in:
45 ```

50 documentation if you set the option `HAVE_DOT` in the file `src/doc/Doxyfile` to
51 `NO`. However, then there will be no graphs generated by `doxygen`.

62 # Purpose
63 
64 This repository provides applications for multiple purposes. Depending on your
65 task at hand you may only need individual parts and do not need to compile all
66 components (see components table below). As a new user you may find yourself in
67 one of these three possible scenarios:
68 
69 ## Prediction with existing neural network potential
70 
71 If you have a working neural network potential setup (i.e. a settings file with
72 network and symmetry function parameters, weight files and a scaling file)
73 ready and want to predict energies and forces for a single structure you only
74 need these components:
75 
76 - [libnnp](libnnp.md)
77 - [nnp-predict](nnp-predict.md)
78 
79 ## Molecular dynamics simulation
80 
81 Similarly, if you have a working neural network potential setup and would like
82 to run an MD simulation with an external MD software (so far only LAMMPS is
83 supported), these components are required:
84 
85 - [libnnp](libnnp.md)
86 - `libnnpif`
87 - [pair_style nnp](if_lammps.md)
88 
89 ## Training a new neural network potential
90 
91 To train a completely new neural network potential the following parts are required:
92 
93 - [libnnp](libnnp.md)
94 - `libnnptrain`
95 - [nnp-scaling](nnp-scaling.md)
96 - [nnp-train](nnp-train.md)
97 
98 Additional, though not strictly required tools, are also quite useful:
99 
100 - [nnp-comp2](nnp-comp2.md)
101 - [nnp-convert](nnp-convert.md)
102 - [nnp-dataset](nnp-dataset.md)
103 - [nnp-dist](nnp-dist.md)
104 - [nnp-norm](nnp-norm.md)
105 - [nnp-prune](nnp-prune.md)
106 - [nnp-select](nnp-select.md)
107 - [nnp-symfunc](nnp-symfunc.md)

111 ## Code structure
112 
113 This package contains multiple components with varying interdependencies and
114 dependencies on third-party libraries. You may not need to build all
115 components, this depends on the intended use. The following table lists all
116 components and their respective requirements (follow the links for more
117 information).
118 
119 | Component | Location | Requirements | Function |
120 | ------------------------------- | ------------------------------- | -------------------------- | ---------------------------------------------------- |
121 | [libnnp](libnnp.md) | `src` | C++98 compiler (icpc, g++) | NNP core library (NN, SF, Structure, ...) |
122 | libnnpif | `src` | `libnnp`, MPI | Interfaces to other software (LAMMPS, ...) |
123 | libnnptrain | `src` | `libnnp`, MPI, GSL, Eigen | Dataset and training routines (Kalman, ...). |
124 | [nnp-convert](nnp-convert.md) | `src/application` | `libnnp` | Convert between structure file formats. |
125 | [nnp-cutoff](nnp-cutoff.md) | `src/application` | `libnnp` | Test speed of different cutoff functions. |
126 | [nnp-dist](nnp-dist.md) | `src/application` | `libnnp` | Calculate radial and angular distribution functions. |
127 | [nnp-predict](nnp-predict.md) | `src/application` | `libnnp` | Predict energy and forces for one structure. |
128 | [nnp-prune](nnp-prune.md) | `src/application` | `libnnp` | Prune symmetry functions. |
129 | [nnp-select](nnp-select.md) | `src/application` | `libnnp` | Select subset from data set. |
130 | [nnp-symfunc](nnp-symfunc.md) | `src/application` | `libnnp` | Symmetry function shape from settings file. |
131 | [nnp-comp2](nnp-comp2.md) | `src/application` | `libnnptrain` | Compare prediction of 2 NNPs for data set. |
132 | [nnp-dataset](nnp-dataset.md) | `src/application` | `libnnptrain` | Calculate energies and forces for a whole data set. |
133 | [nnp-norm](nnp-norm.md) | `src/application` | `libnnptrain` | Calculate normalization factors for data set. |
134 | [nnp-scaling](nnp-scaling.md) | `src/application` | `libnnptrain` | Calculate symmetry function values for data set. |
135 | [nnp-train](nnp-train.md) | `src/application` | `libnnptrain` | Train a neural network potential. |

140 ## The simple way using the master makefile
141 
142 A master makefile is provided in the `src` directory which provides targets for
143 all individual components (except for the LAMMPS [pair_style nnp](if_lammps.md)).
144 For instance, compiling the interface library `libnnpif` requires only to type
145 ```
146 cd src
147 make libnnpif-shared
148 ```
149 This will automatically build a shared version (for dynamic linking) of the
150 required library `libnnp` and the requested `libnnpif`. Similarly, a static
151 build is done like this:
152 ```
153 make libnnpif-static
154 ```
155 To build everything (all libraries and tools) type:
156 ```
157 make shared
158 ```
159 or
160 ```
161 make static
162 ```
163 Compiled binaries will be copied to the `bin` path (relative to the root
164 directory), whereas libraries can be found in the `lib` folder. To clean up
165 individual components or everything use one of these makefile targets
166 ```
167 make clean
168 make clean-doc
169 make clean-libnnpif
170 ```
171 Have a look at the main makefile in `src` for all available build
172 targets.
173 
174 Currently the build process has been tested with two different compilers, the
175 GNU compiler g++ 5.4 (`gnu`) and the Intel compiler 17 (`intel`). It is
176 possible to switch between them via the `COMP` variable, e.g.
177 ```
178 make libnnp COMP=intel
179 ```
180 If you need to change compiler variables and paths have a look at the
181 corresponding makefiles containing global build parameters:
182 ```
183 src/makefile.gnu
184 src/makefile.intel
185 ```
186 You can also create new parameter makefiles based on the above and change the
187 file name suffix according to your target:
188 ```
189 src/makefile.target
190 make libnnp COMP=target
191 ```
192 By default the makefile will use 4 processors for compiling multiple files at
193 once, this behaviour can be overriden with the CORES switch, e.g. to use 8
194 cores:
195 ```
196 make libnnp CORES=-j8
197 ```
198 @warning
199 Do not use the `-j` switch for the master makefile, this may mess up
200 the compilation order.
201 
202 ## Individual component makefiles
203 
204 It is also possible to invoke individual makefiles for each component manually.
205 Just switch to the corresponding folder and use `make shared` or `make static`.
